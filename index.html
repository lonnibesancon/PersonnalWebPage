About Me

Since November 2014 I am a PhD Student at Inria and Limsi/CNRS at Université Paris Sud.
I am supervised by Tobias Isenberg from Inria and Mehdi Ammi from LIMSI/CNRS.
I am currently working on finding a continuum between all the interaction and visualization devices we currently have in order to propose better scientific visualization software and tools. I aim at finding and combining advantages of different interaction mediums in order to allow researchers to visualize and gain insights in one environment and be able to transfer these insights to an other environment which is better suited for the remaining work.
My work includes collaborative sense-making of data, interaction techniques (tangible, tactile...) and classical scientific visualization methods and algorithms.

Haptic Pong

Through a course on Haptic Interactions, I developped with a classmate a version of the classic Pong game with haptic feedback. Basically, an haptich device serves as a medium for both input (through arm movement) and output(through forces indicating the ball's direction and speed when hitting the racket).
The project has been coded in C thanks to a haptic library and the graphics were done in OpenGL.

IRT SystemX

Research Engineer Intern• March 2014 - September 2014

Design, Implementation and Evaluation on driving simulateur of the interface of semi-autonomous cars.
Partners: Renault, Alstom, Valeo, M3Systems, SysNav, CAE
Design of the visual and sound HCI (Human Computer Interface) with the help of a sound designer. The HCI specifications were made thanks to a user-centered design as well as a state of the art and an analysis of the existing prototypes. Implementation of the HCI and its new components with engineers from Valeo onto a driving simulator. Evaluation of the HCI concept in order to refine the HCI specifications.

Thales Air Systems

Software Developper Intern • June 2013 - September 2013

Development of an application which also involved Ecole Centrale Paris in a research project, so as to define a method to help generating and evaluating new antennas' concepts using Bayes Nets, CSPs, and several Java APIs.
The application developed uses Bayes Nets so as to represent and compute all the requirements an antenna should respect. The program allows to add requirements, possible values for different components and/or performance constraints and the user can ask the program to solve the problem which will be done thanks to a fast algorithm.
The application itself allows engineers from Thales to easily create Bayes Nets without ever manipulating them so that they do not require any Bayes Net background to solve their complex problems. Hence, getting the ideal architecture of a new antenna is easier since engineers only need to formulate their problem ( constraints, performance, requirements, available pieces ... ). The resolution of the problem itself is done by the program.
Delivered :

  • Java program with an intuitive interface
  • Complete documentation on the program and some examples
  • Teaching sessions on how to think the conception problems differently so the program can be used
